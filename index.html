(F.A.Q) Pertanyaan yang sering diajukan?

Bisakah Anda menjamin setiap bagian dari web bisa di convert dengan benar??

Sayangnya kami tidak dapat :( Kami membuat perayap dan peretas dari awal dan melakukan pekerjaan yang cukup bagus dalam mencari dan merayapi setiap sudut situs tertentu. Namun, karena perayap tidak memproses Javascript, ada peluang bagus bahwa sumber daya dimuat secara dinamis oleh Javascript tidak akan dimasukkan dalam kloningan.

Bagaimana tautan ditangani? Apakah semua tautan ditulis ulang? Dapat membuka kloning di komputer saya setelah mengunduhnya?

Selama proses kloning, SEMUA LINK ditulis ulang dan diubah menjadi URL relatif, ini memastikan situs kloning akan berfungsi dengan baik di komputer seperti pada server web.

Apa artinya bila proses kloning telah "habis waktunya"?

Sebagai tindakan keamanan, ada jumlah detik maksimum yang ditetapkan agar proses kloning dibiarkan berjalan. Ketika batas ini tercapai, proses akan berhenti. Kemajuan yang dibuat sejauh ini akan menghemat meskipun, dan Anda dapat melanjutkan dengan proses pada waktu tertentu dengan mengklik tombol continue () untuk klon itu. Batas waktu habis dapat diatur untuk masing-masing situs dan default diatur ke 1800 detik.

Apakah saya diizinkan untuk mengkloning situs web orang lain?

Sejauh yang kami tahu, itu tidak ilegal atau dilarang oleh hukum untuk mengkloning situs web orang lain. Dikatakan, karena sebagian besar situs web dilindungi oleh undang-undang hak cipta, Anda tidak diperbolehkan mengkloning situs acak dan menggunakan asetnya di proyek Anda sendiri tanpa izin dari pemilik situs.

Bagian-bagian tertentu kloning saya tampaknya tidak berfungsi dengan baik, tahu mengapa?

Paling sering ini disebabkan oleh konten dinamis; tempat Javascript mengambil data dari server dan kemudian memperbarui situs. Karena Skrip Ajaib tidak mengkloning skrip sisi server seperti PHP, pembaruan dinamis tidak akan berfungsi di sebagian besar situasi :(

Apakah https didukung ??

Saat ini https tidak didukung; jika ada permintaan yang cukup, kami dapat menambahkan dukungan untuk http yang aman di masa mendatang.

Bagaimana dengan sumber daya seperti gambar dan file font yang ditautkan dari file CSS eksternal?

Selama semua file ini disertakan mengikuti konvensi umum (@import untuk stylesheet tambahan, @ font-face untuk font dan url () sintaks untuk gambar), setiap sumber daya yang terhubung akan diambil dan dirayapi. Kami telah melakukan pengujian ekstensif dan menemukan apa pun yang ditautkan dari file CSS yang dirayapi dan diindeks dengan benar. Jika Anda menemukan kejadian di mana ada sesuatu yang terlewatkan, harap beri tahu kami dan kami akan men-tweak crawler untuk memastikan sumber daya yang diambil juga.

Apakah itu menghemat untuk membatalkan proses kloning saat itu masih kloning?

Ya, tentu saja. Ketika Anda membatalkan proses kloning, skrip akan mengingat kemajuan yang telah dibuat sejauh ini sehingga Anda melanjutkan proses kloning pada saat tertentu dengan mengklik tombol Continue untuk kloning itu.

Sambil melihat kemajuan kloning, saya melihat banyak angka. Apa artinya ini?

Angka-angka ini menunjukkan jumlah memori yang digunakan oleh skrip perayapan. Ini hanya untuk melacak berapa banyak memori yang digunakan skrip dan pastikan tidak mengkonsumsi terlalu banyak.
